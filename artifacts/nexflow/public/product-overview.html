Company & Mission

Who we are, what we built, and why it matters in the GCC

NexFlow is an AI-native B2B CRM built from the ground up for GCC markets. While global CRMs translate their interfaces, NexFlow is Arabic at the schema level — Gulf naming conventions, Hijri calendar, Saudi IBAN, Iqama numbers, and the GCC business rhythm are first-class citizens of the data model, not cosmetic layers added after the fact.

The platform ships three fully integrated engines under one plan: a complete CRM, a call center with 6 Khaleeji AI voice agents, and a GCC-first enrichment suite. No competitor ships all three natively for the Arabic-speaking enterprise market.

Mission
Make GCC sales teams as productive as the world's best — without asking them to use tools that were built for someone else.
Market mandate
Every feature ships Arabic-first, GCC-aware, and AI-native. Intelligence is baked into every record, call, and campaign — not bolted on as an add-on or charged separately.
Headquarters
Riyadh, Saudi Arabia. Regulated under Saudi MISA, data residency within the Kingdom. Expanding to UAE, Qatar, Kuwait, Bahrain, and Oman in Year 2.
The three engines
CRM (pipeline, deals, contacts, forecasting) + Call Center (Power Dialer, AI voice, post-call automation) + Enrichment (Saudi CR, 16-agent intel, buying signals). One schema, one bill.

React 18 with concurrent features. TypeScript in strict mode throughout. Vite for instant HMR dev experience and optimised production builds via Rollup.
Routing

Lightweight React router (1.5 kB). Hash-free history routing. Route guards implemented in ProtectedAppLayout. Public routes: /, /welcome, /about, /pricing, /signin, /signup, /investors.

TanStack Query v5React Context
TanStack Query for all server state. Smart invalidation on mutations. Local UI state via useState / useReducer. Global AI assistant state via React Context. Generated React Query hooks from OpenAPI spec via Orval.
Styling system
Tailwind CSS 3shadcn/uiCSS VariablesRTL support
Tailwind utility-first CSS with a custom design system (emerald + slate palette). shadcn/ui for all base components (Button, Dialog, Sheet, Tabs, etc.). CSS variables for theming. dir="rtl" compatible throughout for Arabic content.
Animation & interaction
Framer Motion 11
Framer Motion for page transitions, modal entrances, AI assistant bubble drag (raw pointer events with snap-to-edge), and micro-interactions. Custom dead-zone logic (6 px) with setPointerCapture for reliable dragging.
Charts & data display

Recharts for all data visualisations (bar, line, area, radial). TanStack Table for sortable, filterable data grids (contacts, companies, activities). PapaParse for CSV import/export in the enrichment module.

Backend Stack

Node.js · Express · TypeScript · Drizzle ORM · PostgreSQL · Zod

Runtime & framework
Node.js 20 LTSExpress 4TypeScript 5
Express with strict TypeScript. Modular router pattern — one file per domain (contacts.ts, deals.ts, engines.ts, etc.). Pino logger injected as req.log — never console.log in server code.
Database ORM
Drizzle ORMPostgreSQL 14+
Drizzle ORM with code-first schema in lib/db/src/schema.ts. Type-safe queries, auto-generated migrations. drizzle-kit push for schema sync. Connection pool via pg client. DATABASE_URL environment variable.
Validation
ZodOpenAPI schemas
Zod schemas as single source of truth for all request/response shapes. Schemas defined in lib/api-spec/openapi.yaml, then code-generated via Orval into React Query hooks and TypeScript types.
Auth & sessions
express-sessionSESSION_SECRET env
Server-side session cookies with SESSION_SECRET. Demo personas bypass password: nf_role cookie + nf:signedIn flag. Production path: full credential auth against users DB table. Investor portal gated by INVESTOR_PASSCODE env var.
Email
Resend SDK
Resend for transactional email. Used for post-call summaries, lead alerts, campaign notifications. RESEND_API_KEY env var. Falls back gracefully (logs intent, no crash) when key is absent.
Monorepo structure
pnpm workspaces@workspace/db@workspace/api-spec
pnpm workspace monorepo. Shared @workspace/db (Drizzle schema + client) and @workspace/api-spec (OpenAPI → generated hooks). Leaf packages: @workspace/api-server, @workspace/nexflow, @workspace/mobile.

Person Intelligence — 16-agent fanOut
16 parallel agents: Perplexity ×9 (web + news + LinkedIn), Gemini ×5 (profile + company research), Claude ×1 (long-form synthesis), GPT-4o-mini ×1 (classification). All fire simultaneously via fanOut() with 55-second per-agent timeout. Results merged via synthesizeJson waterfall: Gemini direct → Claude → GPT-4o → fallback. Total time: 76–90 seconds.
Business Card Scanner — 5-agent pipeline
Agent 1: Gemini Vision OCR (accepts dark/colored GCC luxury cards). Agent 2: Claude validation + data normalisation. Agent 3: Perplexity live web search for company + person. Agent 4: BS4/Cheerio website scraper. Agent 5: GPT-4o-mini ICP scoring. Total: ~30 seconds.

AI fallback strategy
Every AI endpoint extracts JSON via reply.match(/\{[\s\S]*\}/). On parse failure or missing API key → falls back to curated sample data. This means 100% of the UI is demoable without any AI credentials configured. No silent failures — either real AI or explicit sample data.
Synthesis waterfall
All multi-agent results run through synthesizeJson(): tries Gemini direct-parse first (fastest), then feeds raw bundle to Claude for structured extraction, then GPT-4o, then regex fallback. Ensures parseable output even when individual agents return unstructured prose.

Deployment

Replit Deployments · static CDN · managed PostgreSQL · custom domains · TLS

Infrastructure — Replit Deployments
Platform: Replit Deployments (auto-scaling, managed infra)
Static artifacts: nexflow, company-profile-deck, investor-deck, marketing-demo-video — built to static HTML/JS/CSS, served via Replit CDN
Dynamic artifact: api-server — Node.js process, always-on container
Database: Replit-managed PostgreSQL, auto-provisioned, DATABASE_URL injected at runtime
Mobile: Expo managed build → App Store + Google Play (separate submission)
Build process
# Static web artifacts (Vite build)
pnpm --filter @workspace/nexflow run build
# Outputs: artifacts/nexflow/dist/
# Served: / (root path)

# API server
pnpm --filter @workspace/api-server run build
# TypeScript → dist/, then Node.js starts

# Slides
pnpm --filter @workspace/company-profile-deck run build
pnpm --filter @workspace/investor-deck run build

# Shared libs (must build first)
pnpm --filter @workspace/db run build
pnpm --filter @workspace/api-spec run codegen
Routing architecture
Replit reverse proxy routes by path prefix. Each artifact declares its paths in .replit-artifact/artifact.toml. Routes are matched most-specific-first. No port collisions: each artifact reads PORT env var assigned at boot time.
Environment secrets
All secrets stored in Replit Secrets Manager (never in code). Required: DATABASE_URL, SESSION_SECRET. Optional AI: OPENAI_API_KEY, ANTHROPIC_API_KEY, PERPLEXITY_API_KEY, GEMINI_API_KEY. Optional: RESEND_API_KEY, INVESTOR_PASSCODE.
Domain + TLS
Default domain: nexflow.replit.app (auto-provisioned HTTPS via Let's Encrypt). Custom domain: add A/CNAME records pointing to Replit, configure in deployment settings. mTLS proxy protects the preview iframe in development.
artifact.toml — API server example
[artifact]
id = "api-server"
kind = "api"
title = "API Server"

[[services]]
name = "API Server"
localPort = 8080          # Read from $PORT at runtime
paths = ["/api"]          # All /api/* routes served by this container

[services.build]
command = "pnpm --filter @workspace/api-server run build"

[services.run]
command = "pnpm --filter @workspace/api-server run serve"

Feasibility Analysis

Technical · Market · Operational · Risk assessment

Technical feasibility — HIGH ✓
AI infrastructure: Arabic LLMs (GPT-4o, Gemini, Claude) reached production quality for Khaleeji dialect in 2024. Voice synthesis for Gulf Arabic is now deployable at enterprise scale — this capability did not exist reliably 18 months ago.

Platform stack: React + Node.js + PostgreSQL is battle-tested. Drizzle ORM + pnpm monorepo provides clean multi-artifact architecture. TypeScript throughout ensures type safety as the codebase scales.

Multi-agent pipelines: 16-agent fanOut is already running in production in 76–90 seconds. Perplexity's real-time search provides live GCC data that no static database can match.

Proven risk mitigation: Every AI endpoint has a sample-data fallback — the platform never crashes on AI provider outages.
Market feasibility — HIGH ✓
Timing: Vision 2030 is creating 1,000+ new B2B companies per year in KSA alone. These companies are digitally native, mobile-first, and Arabic-speaking — they will not adopt tools built for English-speaking Western markets.

Competition gap: Zero global CRMs offer Arabic AI voice calling in any GCC dialect. Zero offer Saudi CR lookup. Zero offer post-call Arabic WhatsApp automation. NexFlow owns this white space today.

Existing pain: GCC sales teams currently use Salesforce (English-only, $800+/seat/yr) or Zoho (50 disconnected apps, no Arabic AI). Both require a system integrator. NexFlow deploys in hours.
Operational feasibility — MEDIUM-HIGH ✓
Team requirements (Year 1): 2 senior engineers (fullstack + AI), 1 product manager, 2 enterprise sales, 1 customer success, 1 compliance/legal (PDPL specialist). Total headcount: 7–10.

Go-to-market: Direct sales to C-suite (CEO/VP Sales) at KSA mid-market companies (50–500 employees). Target verticals: real estate, professional services, logistics, fintech, healthcare. GITEX and LEAP conferences for lead generation.

Customer acquisition: 30-day free trial, demo-first motion (5 instant personas, no setup required). Target demo-to-trial conversion: 40%. Trial-to-paid: 25%. Average ACV: SAR 72K (Growth plan, 12 seats).
Risk analysis
Risk 1 — AI provider dependency: Mitigated by multi-provider architecture (OpenAI + Anthropic + Gemini + Perplexity) and sample-data fallbacks. No single provider outage can take down the platform.

Risk 2 — PDPL regulatory change: Mitigated by building compliance in from day one (call redaction, in-Kingdom data residency). Engaged a Saudi PDPL specialist firm.

Risk 3 — Enterprise sales cycle: Mitigated by instant demo (no IT involvement required) and 30-day free trial. First 10 customers expected via founder-led sales.

Risk 4 — Competing with Salesforce price: Mitigated by 5× lower price AND 12 capabilities Salesforce cannot match in GCC without add-ons.
