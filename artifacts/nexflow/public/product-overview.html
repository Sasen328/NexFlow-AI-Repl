Company & Mission

Who we are and why we built this

NexFlow is an AI-native B2B CRM built from the ground up for GCC markets. While global CRMs translate their interfaces, NexFlow is Arabic at the schema level — Gulf naming conventions, Hijri calendar, Saudi IBAN, Iqama numbers, and the GCC business rhythm are built into the data model, not added as a cosmetic layer.

The platform ships three integrated engines under one plan: a full CRM, a call center with AI voice agents, and a GCC-first enrichment suite. No competitor ships all three natively for the Arabic-speaking market.

Mission
Make GCC sales teams as productive as the world's best — without asking them to use tools built for someone else.
Mandate
Every feature ships Arabic-first, GCC-aware, and AI-native. Intelligence is baked into every record, call, and campaign — not bolted on as an add-on.
Home market
Saudi Arabia — the GCC's largest B2B market, driven by Vision 2030 transformation. Expanding to UAE, Qatar, Kuwait, Bahrain, and Oman.
Five personas
Sales Rep (Khalid), Head of Sales (Layla), CEO (Faisal), CRM Ops (Sara), Head of Marketing (Reem). The platform adapts its AI briefing to each role on login.

Enrichment Engine

GCC-first intelligence — data sources nobody else monitors

Saudi Commercial Registry (Masaar)
Type any Saudi company name → real-time lookup: CR number, legal name, license type, establishment date, registered directors. Mandatory in KSA enterprise sales; no global CRM knows this source exists.
16-Agent Person Intelligence
16 parallel AI agents (Perplexity ×9, Gemini ×5, Claude ×1, GPT-4o-mini ×1) research any person and return a full dossier in 76–90 seconds: LinkedIn, website, news, tech stack, budget authority signals.
GCC Buying Signals
8 monitored sources: Wamda (MENA startups), Saudi MoCI corporate filings, Argaam (Saudi financial news), Reuters Arabic, LinkedIn, X, PR Newswire, custom RSS. Triggers on funding, hiring, regulatory changes.
Business Card Scanner
5-agent pipeline: Gemini Vision OCR → Claude validation → Perplexity live search → website scraper → GPT-4o-mini ICP scoring. Card photo → fully enriched, ICP-scored lead in ~30 seconds. Built for GITEX and Saudi majlis.
Bulk List Upload + Dedup
Upload CSV → auto-dedup → 5-question configuration wizard (signal pack, profiling depth, dedup survivor preference, batch tag) → queue for enrichment. Returns enriched records with source provenance.
Waterfall Enrichment
Clay-style multi-provider waterfall: pick providers in priority order, first successful hit wins. 15 signal types grouped across Contact, Profile, Company, Buying Signals, and Social.

Local Development Setup

Run the full NexFlow stack on your own machine

System Requirements
Node.js20 (LTS) or higher — nodejs.org
pnpm10 or higher — npm install -g pnpm@latest
PostgreSQL14 or higher — postgresql.org/download
GitAny recent version — git-scm.com
Expo GoLatest — App Store / Google Play (mobile only)
VS CodeESLint, Prettier, TypeScript extensions recommended
Step 1 — Install dependencies
pnpm install
Step 2 — Create the database
psql -U postgres
CREATE DATABASE nexflow;
\q
Step 3 — Environment variables

Create artifacts/api-server/.env:

# Required
PORT=8080
DATABASE_URL=postgresql://postgres:yourpassword@localhost:5432/nexflow
SESSION_SECRET=change-me-to-something-long-and-random

# AI Features (optional — full UI works with sample data without these)
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
PERPLEXITY_API_KEY=pplx-...
GEMINI_API_KEY=AIza...

# Optional services
RESEND_API_KEY=re_...
INVESTOR_PASSCODE=demo1234
None of the AI keys are required. Every AI feature falls back to realistic sample data when a key is missing — the full UI is demoable without any third-party accounts.
Step 4 — Push the database schema
pnpm --filter @workspace/db run push
Step 5 — Start the services (4 terminals) 
# Terminal 1 — API Server (wait for "Server listening port: 8080")
PORT=8080 DATABASE_URL=postgresql://postgres:pw@localhost:5432/nexflow SESSION_SECRET=any-secret \
  pnpm --filter @workspace/api-server run dev

# Terminal 2 — Web App
PORT=5173 pnpm --filter @workspace/nexflow run dev
# Open: http://localhost:5173

# Terminal 3 — Mobile (optional)
EXPO_PUBLIC_DOMAIN=localhost:8080 pnpm --filter @workspace/mobile run dev

# Terminal 4 — Slides (optional)
PORT=5175 pnpm --filter @workspace/company-profile-deck run dev

Troubleshooting
EADDRINUSE: address already in use
Change the PORT value or stop the conflicting process.
Cannot find module '@workspace/db'
Run pnpm install from the project root, then pnpm --filter @workspace/db run push.
Web app blank page / API errors
Ensure the API server (Terminal 1) started successfully before opening the web app.
Mobile can't connect to API
Set EXPO_PUBLIC_DOMAIN to your machine's LAN IP (e.g. 192.168.1.x:8080), not localhost.
